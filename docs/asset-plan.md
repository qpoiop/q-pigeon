# PIGEON PROTOCOL — 3D 에셋 요구 문서 (Asset Requirements)

> 목적: 현재 **절차적 프리미티브**로 그려지는 비주얼을 어디에서 **전용 3D 모델 에셋**으로
> 전환할지, 각 에셋의 스펙·우선순위·사용처·소싱 방향을 정리한다. 구현 착수 전 참조/발주용.
>
> 스코프 결정(합의됨): **게임필/주스는 코드로 진행**. **무거운 라이팅(실시간 그림자·SSAO·
> 베이크드 GI)은 현 컨셉 대비 과잉이라 보류.** 따라서 에셋 방향은 **스타일라이즈드 저폴리
> glTF**(핸드 실루엣 + 버텍스 컬러 / 토ون), **PBR·노멀맵 최소화**로 현 모더니스트 톤을 유지한다.

---

## 1. 현재 상태 (baseline)

- 모든 캐릭터·환경·소품이 코드에서 THREE 프리미티브 조합으로 생성됨
  (`src/game/birds.ts`의 `makeBird`, `src/game/engine.ts`의 지오메트리 생성부).
- 머티리얼: `MeshLambertMaterial` 단색. **텍스처/노멀/러프니스 맵 없음.**
- 애니메이션: `animBird`의 **절차적 레이어**(날개 퍼덕임, 고개 보브, 바디 린, 크라우치 블렌드).
- 라이팅: 디렉셔널 sun + 앰비언트. 그림자 품질 낮음(의도).

## 2. 아트 디렉션 제약 (에셋 발주 시 필수 준수)

| 항목 | 지침 |
|---|---|
| 스타일 | 스타일라이즈드 저폴리. 플랫/토ون 셰이딩 전제. 포토리얼 금지 |
| 머티리얼 | 버텍스 컬러 또는 소형 팔레트 텍스처. **PBR 세트(알베도+노멀+러프+메탈) 불필요** |
| 팔레트 | 모더니스트: 종이색(#f3f2f2 계열) 바탕, 잉크 블랙, 액센트 레드(#ec3013/#d64525) |
| 폴리 예산 | 히어로 2–4k tri, 소품 300–800 tri, 환경 조각 인스턴싱 전제 저폴리 |
| 스케일 | 현 월드 단위 유지(새 반경 ≈ 0.46, 벽 높이 등 `levels.ts` 기준) |
| 포맷 | **glTF 2.0 (.glb), Draco 압축**. 텍스처는 KTX2/WebP 권장 |

## 3. 에셋 인벤토리 (전환 대상)

### 🔴 P0 — 히어로 (항상 화면, 최대 임팩트)

#### A1. 요원 새 3종 — 비둘기 / 까치 / 부엉이
- **현재**: `makeBird`(구체 바디 + 콘 부리 + 구체 눈 + 구체 날개 + 박스 꼬리 + 실린더 다리), 종별 파츠 분기(까치=긴 꼬리, 부엉이=귀털·큰 눈, 비둘기=목).
- **필요 에셋**: 리그드 캐릭터 3종. **공용 스켈레톤 1개** + 종별 메시/실루엣.
- **스펙**: .glb+Draco, 2–4k tri/종, 스켈레탈 리그(스파인·헤드·윙L/R·레그L/R 최소).
- **애니메이션 클립(필수)**: `idle`, `walk`, `run`, `crouch_idle`, `crouch_move`, `dash`, `alert`(발각 반응). 루프/원샷 명시.
- **데이터 연결**: `src/data/characters.ts`의 `pigeon|magpie|owl`, 팔레트(`pal`) 매핑 유지.
- **사용처**: 플레이어(1인칭 아님, 탑다운 추적 카메라), 원격 피어(코옵 유령).

#### A2. 경비 새 (guard)
- **현재**: `makeBird('guard')` — 어두운 팔레트 변형(#2b2825 등) + 헬멧/바이저/베레 파츠.
- **필요 에셋**: **A1 공용 리그 재사용** + 경비 스킨 + 장비(모자·바이저·완장).
- **애니메이션**: A1 클립 재사용 + `patrol_walk`, `search_lookaround`, `chase_run` 포즈 강조.
- **사용처**: 적 AI(순찰/추격/수색/미끼). 상태별 시야콘은 코드 유지(에셋 아님).

### 🟠 P1 — 환경 킷 (세계관 상승 폭 큼)

#### B1. 모듈러 시설 킷
- **현재**: 벽 = `BoxGeometry`, 은폐 구역 = `PlaneGeometry`+edges, 바닥 = `PlaneGeometry`.
- **필요 에셋(모듈)**: 벽 패널 · 코너 · 펜스/철망 · 바리케이드 · 크레이트 · 드럼통 · 감시탑 · 게이트 · 파이프/덕트 · 저식생(관목/화분). **인스턴싱 전제 저폴리.**
- **스펙**: .glb, 조각당 300–1500 tri, 그리드 스냅 가능한 정렬 원점.
- **데이터 연결**: `levels.ts`의 `walls`(Bounds), `covers` 좌표에 킷 조각을 배치하는 매핑 레이어 필요(신규).
- **사용처**: 전 레벨 배경/차폐. 은폐 구역은 "완전 은신" 판정 유지 → 시각적으로 그늘/구조물로 표현.

#### B2. 바닥/구역 데칼 세트
- **현재**: 종이색 평면 + 아우터 평면.
- **필요 에셋**: 바닥 타일 텍스처(콘크리트/격자/경계선), 구역 마킹 데칼(순찰로·위험구역).
- **스펙**: 타일러블 텍스처(WebP/KTX2), 데칼 알파. **모델 아님.**

### 🟡 P2 — 목표/소품 (작지만 아이코닉)

| ID | 요소 | 현재 | 필요 에셋 | tri | 애님 |
|---|---|---|---|---|---|
| C1 | 마이크로필름(목표) | `Octahedron`+`Torus` | 필름릴/캐니스터 모델 | ~500 | 회전(코드) |
| C2 | 미끼(decoy) 픽업+투척물 | `Box`+stripe cube+ring | 소형 디코이 장치(삐삐/미니버드) | ~400 | 부유·점멸(코드) |
| C3 | 연막(smoke) 픽업 | `Sphere` | 연막 캐니스터 모델 | ~400 | — |
| C4 | 탈출 구역 마커 | Plane+Edges(적색) | 바닥 발광 데칼 or 유지 | — | 펄스(코드) |

- **데이터 연결**: C1 = `films`, C2/C3 = `ItemType('decoy'|'smoke')`(`levels.ts`).

### 🟢 절차 유지 (에셋 불필요 — 발주 제외)
- 시야콘(`CircleGeometry`+알파) — 셰이더로 유지가 정답
- 파티클/디버리(소형 박스), 스모크 퍼프 FX — **텍스처 시트/셰이더**로 업그레이드하되 모델 아님
- 목표 화살표, 경보(!) 스프라이트, 이름표 — 캔버스 UI 유지
- 미니맵 — 2D 캔버스 유지

## 4. 파이프라인 요구사항 (구현 측)

- **로더 도입**: three `GLTFLoader` + `DRACOLoader`(+ KTX2 `KTX2Loader` 텍스처 시).
- **번들 예산**: 현 프리캐시 ~710KB. 모델은 **지연 로딩 + Draco**로 관리. 히어로 셋 우선 프리로드,
  환경 킷은 스테이지 진입 시 로드.
- **애니메이션 통합**: 스켈레탈 클립을 `AnimationMixer`로 재생하되, 기존 `animBird`의 절차 레이어
  (고개 글랜스·바디 린)는 **애디티브로 유지**해 생동감 보존.
- **머티리얼**: `MeshToonMaterial` 또는 현 `MeshLambert`+버텍스컬러. 그림자맵 해상도/컷 유지(무거운 라이팅 X).
- **인스턴싱**: 환경 킷 반복 조각은 `InstancedMesh`로 드로우콜 억제.

## 5. 소싱 방향 (코드 불가 — 외부 제작/구매)

| 대상 | 옵션 |
|---|---|
| 히어로 새 리그 3종 + 경비 | 커스텀 제작(권장) 또는 Quaternius 스타일라이즈드 애니멀 팩 개조 |
| 환경 킷 | Kenney "Modular"/"Survival", Synty POLYGON, Quaternius 로우폴리 — 톤 정합성 확인 후 |
| 소품(필름/캐니스터) | 커스텀 소량 제작 |
| 텍스처/데칼 | Kenney 텍스처, 자체 제작 |

## 6. 추천 착수 순서

1. **T1 게임필/주스** (코드, 에셋 무관) — 즉시 착수 가능
2. **glTF 파이프라인 배선** — placeholder 모델 1개로 로더/Draco/믹서 검증
3. **A1·A2 히어로 새 교체** — 최대 임팩트
4. **B1 환경 킷** — 세계관 상승
5. **C1–C3 소품** 순차
